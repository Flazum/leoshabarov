Bases & Adics

A clear, page-by-page walk from ordinary bases to the p-adic world (no animations, just crisp prose).

Why 2 + 2 can be 10 or 100

In base 10, 2+2=4. In base 4, the next symbol after 3 is 10, so 2+2=10 (base 4). In base 2, 2+2=100 (binary). In base 3, 2+2=11. The symbols and carry threshold change with the base; the quantity doesn’t.

What a base is

  • Base b uses digits 0,1,2,…,b−1.
  • Place values are b⁰, b¹, b², … to the left.

Example (base 7): 351_7 = 3·7² + 5·7 + 1 = 183.

Reading any base back to decimal

For digits a_k…a₂a₁a₀ in base b:
a₀·b⁰ + a₁·b¹ + … + a_k·bᵏ.

Example: 1004_7 = 343 + 4 = 347.

Converting a decimal integer to base b

  1. Divide by b, record remainder (least-significant digit).
  2. Replace by ⌊n/b⌋ and repeat to zero.
  3. Read remainders in reverse.

Example: 347 → 1004_7.

Converting fractional parts

  1. Multiply fraction by b.
  2. Take the integer part as the next digit.
  3. Repeat with the new fractional part.

Example: 0.625₁₀ = 0.101₂. A fraction terminates in base b iff the reduced denominator’s primes all divide b.

Adding and multiplying in other bases

Same column rules as base 10; only the carry threshold changes from 10 to b.

Example (base 7 addition): 256₇ + 345₇ = 634₇.

Quick base facts

  • Divisible by b ⇔ last digit is 0 in base b.
  • Last k digits determine the number mod bᵏ.
  • Changing base changes symbols, not quantity.

From bases to adic thinking

In p-adics we judge closeness by agreeing on more trailing base-p digits. Infinite expansions run unbounded to the left:
…d₃d₂d₁d₀ = a₀ + a₁p + a₂p² + …

Example: in 10-adics, …9999 = −1; in 2-adics, …1111 = −1.

Building b-adic integers

Either as compatible remainders xₙ (mod bⁿ) with xₙ₊₁≡xₙ, or as digits a₀+a₁b+a₂b²+…. Taking all levels simultaneously encodes one b-adic integer.

Prime bases and the p-adic absolute value

v_p(n) = exponent of p in n; |n|_p = p^{-v_p(n)} (and |0|_p=0). Non-Archimedean: |x+y|_p ≤ max(|x|_p,|y|_p).

Completing ℚ to get ℚₚ and ℤₚ

Complete the rationals under d_p(x,y)=|x−y|_p to obtain ℚₚ. The closed unit ball (elements with |·|_p ≤ 1) is ℤₚ. Every x∈ℚₚ is p^k·u with k∈ℤ, u∈ℤₚ a unit (a₀≠0 mod p).

Series that converge p-adically

1+p+p²+… = 1/(1−p) since |p|_p<1. In 2-adics, that sum is −1. In 10-adics, 1+10+100+… = −1/9.

Doing arithmetic in ℤₚ

Column addition/multiplication work; each column sees only finitely many carries. A unit (first digit nonzero mod p) has a multiplicative inverse found by successive lifting.

Example (inverse of 3 in 5-adics): digits start 2,3,1,… giving 2 + 3·5 + 1·5² + ….

Hensel’s lemma (idea)

If f(a)≡0 (mod p) and f′(a)≠0 (mod p), there’s a unique x∈ℤₚ with x≡a (mod p) and f(x)=0. Newton iteration in ℤₚ performs the lift.

What b-adics are when b isn’t prime

If b=∏ p_i^{e_i}, then ℤ_b ≅ ∏ ℤ_{p_i} (Chinese Remainder intuition).

Worked mini-examples

  • In 2-adics, 1+2+4+8+…=−1 (partial sums differ from −1 by a power of 2).
  • Write −1 in 10-adics: …999999.
  • Inverse of 7 in 2-adics begins 1+2+0·2²+… (lift mod powers of 2).
