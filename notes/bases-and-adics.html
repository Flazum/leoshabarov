Bases and Adics

2 + 2 doesn’t have to look like 4. Change your glasses—the base—and the same quantity wears a new outfit.

Place value in one breath

We usually live in Base 10. Digits are 0–9. The number 1234 really means 1×10³ + 2×10² + 3×10¹ + 4×10⁰. Each step left multiplies by 10.

If you swap 10 for some other base b, the idea is identical: places are powers of b, digits run from 0 to b−1.

Switching bases

Example: convert 1234 (Base 10) to Base 2. Divide by 2 repeatedly and log the remainders from last to first.

1234→617 r0 →308 r1 →154 r0 →77 r0 →38 r1 →19 r0 →9 r1 →4 r1 →2 r0 →1 r0 →0 r1

This writes as 10011010010₂. If you expand it as powers of 2, you get back 1234₁₀.

Same quantity, different costumes: 2+2 is 4₁₀, but also 100₂, 10₄, 11₃.

Other favorites: Base 16 (hex) for compact encoding, and even exotic complex bases like −1+i, which can represent every integer without a minus sign.

Why bases are useful

Bases can reveal structure. Try powers of 5 in Base 10:

5, 25, 625, 390625, 152587890625, 23283064365386962890625, 542101086242752217003726400434970855712890625

The tail digits stabilize and lengthen. If you kept going forever, you’d approach a number that doesn’t change when you square it: x²=x. Beyond the usual 0 and 1, there’s a new kind of solution hiding in a new number system.

Enter the adics

Adic numbers flip our sense of “which way is infinite.” In Base 10-adics, numbers can stretch infinitely to the left. Write …99999. Add 1:

…99999 + 1 = 0

So …99999 = −1. Do normal algebra and it stays consistent:

…99998 = −2  …99997 = −3  …33333 = −⅓

Wild, but legal here. Adics give you extra solutions and new ways to complete patterns.

One caveat: in the 10-adics there can be zero divisors-like behavior, so familiar rules need careful restatement in the proper adic setting. The payoff is a system that makes certain congruences and limits feel natural.
