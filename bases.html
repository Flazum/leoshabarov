What base are we in?

2 + 2 doesn’t always equal 4. Depending on the base, it can be 10, 11, or 100. It’s all about the place-value system you choose.

Positional notation

We usually work in Base 10. The digits are 0–9, and each place is a power of 10. For example, 1234 expands to 1×10³ + 2×10² + 3×10¹ + 4×10⁰.

Changing the base

In Base b, the allowed digits are 0 through b−1, and places are powers of b. To convert a Base-10 number to Base-2 (binary), repeatedly divide by 2 and record remainders from last to first.

1234 ÷ 2 → r0 → 617
617 ÷ 2 → r1 → 308
308 ÷ 2 → r0 → 154
154 ÷ 2 → r0 → 77
77 ÷ 2 → r1 → 38
38 ÷ 2 → r0 → 19
19 ÷ 2 → r1 → 9
9 ÷ 2 → r1 → 4
4 ÷ 2 → r0 → 2
2 ÷ 2 → r0 → 1
1 ÷ 2 → r1 → 0

This yields 10011010010₂. Expanding verifies the value:

1×2¹⁰ + 0×2⁹ + 0×2⁸ + 1×2⁷ + 1×2⁶ + 0×2⁵ + 1×2⁴ + 0×2³ + 0×2² + 1×2¹ + 0×2⁰ = 1234₁₀

So does 2 + 2 = 4?

Yes—in Base 10. But the same quantity looks different in other bases:

  • Base 2: 10₂ + 10₂ = 100₂
  • Base 3: 2₃ + 2₃ = 11₃
  • Base 4: 2₄ + 2₄ = 10₄

Common and unusual bases

  • Base 10: everyday decimal
  • Base 2: binary in digital systems
  • Base 16: hexadecimal for compact numeric encoding
  • Base −1 + i: a complex base that can represent every integer without a minus sign and has surprising links to fractal-like patterns
